# 页面原型说明

## 导航结构

| 一级模块 | 页面 | 主要角色 |
| --- | --- | --- |
| 工作台 | 首页看板 | 全部内部角色 |
| 订单 | 订单列表、订单详情、新建订单 | 销售、设计、生产、财务、管理员 |
| 设计 | 设计任务列表、设计任务详情 | 设计、管理员 |
| 生产 | 生产安排列表、生产安排详情 | 生产、管理员 |
| 客户 | 客户列表、客户详情 | 销售、财务、管理员 |
| 财务 | 发票审批、付款申请 | 财务、管理员 |
| 售后 | 售后申请列表、售后详情 | 销售、售后、管理员 |
| 系统 | 用户与权限、店铺管理 | 管理员 |

## 工作台首页

目标：让不同角色快速看到自己需要处理的事情。

核心内容：

- 今日新增订单数
- 待设计订单数
- 待生产安排订单数
- 待审批发票数
- 待处理售后数
- 最近订单列表
- 我的待办任务

## 订单列表

目标：作为 ERP 的核心入口，支持销售、设计、生产、财务按权限查看订单。

筛选条件：

- 关键词：订单号、平台订单号、客户姓名、手机号
- 店铺
- 订单状态
- 负责人
- 创建时间范围
- 交付时间范围

列表字段：

- 订单号
- 店铺
- 客户
- 金额
- 状态
- 设计状态
- 生产安排状态
- 交付日期
- 负责人
- 创建时间

操作：

- 新建订单
- 查看详情
- 编辑草稿
- 提交订单
- 取消订单

## 新建订单

目标：销售录入订单和定制需求。

字段分组：

- 基础信息：店铺、客户、平台订单号、销售负责人
- 商品信息：商品名称、规格、数量、单价、金额
- 定制信息：设计处理方式、定制说明、尺寸、颜色、客户素材、交付日期、加急标记
- 财务信息：订单金额、已收金额、付款状态
- 附件：客户素材、聊天记录、参考图片

操作：

- 保存草稿
- 提交订单；系统按设计处理方式决定进入设计任务或直接进入生产安排

## 订单详情

目标：集中查看订单完整信息和流程进度。

内容：

- 订单基础信息
- 客户信息
- 订单明细
- 定制需求
- 设计任务
- 生产安排
- 发票申请
- 付款申请
- 售后记录
- 附件
- 操作日志

## 设计任务

目标：设计人员处理定制需求。

列表字段：

- 任务号
- 关联订单
- 客户
- 设计状态
- 设计负责人
- 截止时间
- 更新时间

操作：

- 领取任务
- 上传设计稿
- 标记需修改
- 标记设计确认；确认后进入生产安排

## 生产安排

目标：生产人员确认工厂代工安排。由于生产由工厂代工，生产安排确认后订单即完成。

列表字段：

- 安排号
- 关联订单
- 商品
- 安排状态
- 生产负责人
- 代工工厂
- 计划交付时间

操作：

- 设置生产安排
- 确认生产安排并完成订单
- 记录异常

## 财务页面

发票审批：

- 申请编号、订单号、客户、发票类型、金额、状态、申请人、申请时间
- 操作：通过、驳回、查看附件

付款申请：

- 申请编号、订单号、客户、付款类型、金额、收款方、状态、申请人、申请时间
- 操作：通过、驳回、查看附件

## 售后申请

目标：在订单完成后追加售后问题。

字段：

- 关联订单
- 售后类型
- 问题描述
- 处理方案
- 退款金额
- 状态
- 附件

操作：

- 新建售后
- 受理
- 处理中
- 完成
- 关闭

## 用户与权限

目标：管理员维护内部账号和角色。

内容：

- 用户列表
- 角色分配
- 店铺可见范围
- 账号启用/禁用
- 密码重置

## 设计处理方式设置

目标：管理员维护订单提交时可选的设计处理方式。

字段：

- 选项名称
- 是否需要设计任务
- 默认排序
- 启用状态
- 说明

示例：

- 新订单需要设计
- 老订单修改
- 仅标注
- 续订无需设计
