# 正式开发路线

## 阶段 1：项目骨架

- 创建 Django 后端项目。
- 创建 Vue 3 前端项目。
- 配置 PostgreSQL、Redis、Docker Compose。
- 配置基础登录、角色、权限、菜单。

验收：

- 本地可启动前后端。
- 管理员可以登录后台。
- 数据库迁移可重复执行。

## 阶段 2：订单最小闭环

- 实现店铺、客户、订单、订单明细。
- 实现可自定义的设计处理方式。
- 实现订单草稿、提交、取消。
- 提交订单后按设计处理方式自动分流：需要设计则生成设计任务，无需设计则直接进入生产安排。
- 设计确认后进入生产安排。
- 生产安排确认后订单进入已完成。
- 写入操作日志。

验收：

- 可完整跑通“销售录单 -> 设计处理或跳过设计 -> 生产安排确认 -> 订单完成”。

## 阶段 3：页面与权限

- 实现工作台、订单列表、订单详情、新建订单。
- 实现设计任务和生产安排页面。
- 按销售、设计、生产、财务、管理员限制菜单和按钮。
- 所有列表分页、筛选、排序。

验收：

- 不同角色登录后只能看到自己可操作的页面和按钮。
- 订单列表在模拟数据下保持稳定响应。

## 阶段 4：财务与售后

- 实现发票申请、付款申请、审批通过、审批驳回。
- 实现售后申请、受理、处理、完成、关闭。
- 发票、付款、售后支持附件。

验收：

- 订单详情可查看关联的财务和售后记录。
- 审批流程有日志和状态记录。

## 阶段 5：附件与异步任务

- 实现附件上传、删除、业务关联。
- 接入对象存储配置。
- 实现订单导入、订单导出异步任务。
- 前端显示任务进度。

验收：

- 大文件不拖慢列表页。
- 导入导出时用户仍可正常操作系统。

## 阶段 6：部署与压测

- 编写 Docker 部署脚本。
- 配置 Nginx、HTTPS、静态资源。
- 配置 PostgreSQL 自动备份。
- 准备 5 万到 10 万条模拟订单数据。
- 压测订单列表、订单详情、工作台统计。

验收：

- 云服务器可稳定运行。
- 常用页面在模拟数据下响应正常。
- 数据库备份可恢复。

## 开发顺序

```text
后端模型和迁移
-> 订单状态服务
-> 权限与角色
-> REST API
-> 前端页面
-> 异步任务
-> 部署压测
```

## v1 不做

- 多公司 SaaS 租户化
- 小程序客户端
- 电商平台 API 自动同步
- 复杂库存和采购系统
- 高级生产排程算法
